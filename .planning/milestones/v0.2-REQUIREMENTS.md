# Requirements: efx-skills v0.2.0

**Defined:** 2026-03-05
**Core Value:** Every installed skill knows where it came from and can be opened, updated, or verified against its upstream source.

## v1 Requirements

Requirements for v0.2.0 release. Each maps to roadmap phases.

### Origin & Metadata

- [x] **ORIG-01**: Search results display origin (owner/repo) next to each skill name
- [x] **ORIG-02**: Config.json stores a `skills` array with full metadata per installed skill (owner, skill name, registry type, URL)
- [x] **ORIG-03**: Config.json stores `skills-path` field pointing to central skill storage directory
- [x] **ORIG-04**: Config.json repos entries include `url` field (e.g., `https://github.com/owner/repo`)
- [x] **ORIG-05**: Installing a skill writes its metadata to the `skills` array in config.json
- [x] **ORIG-06**: Uninstalling/removing a skill removes its metadata from the `skills` array in config.json

### Browser Integration

- [x] **BRWS-01**: User can press [o] in search view to open selected skill's source URL in browser
- [x] **BRWS-02**: User can press [o] in manage/provider view to open skill or group URL in browser
- [x] **BRWS-03**: User can press [o] in config view to open registry URL or repo GitHub URL in browser
- [x] **BRWS-04**: Browser open uses `open` (macOS) / `xdg-open` (Linux) for cross-platform support
- [x] **BRWS-05**: Playbooks.com skills open skill-specific URL if available, fallback to playbooks.com domain

### Config Page Redesign

- [x] **CONF-01**: Config page shows registries with friendly names (e.g., "Vercel" instead of "skills.sh") and bold URLs
- [x] **CONF-02**: Config page shows custom repos as `owner    repo-name` format (two columns)
- [x] **CONF-03**: Section label renamed from "Providers" to "Providers search"

### Update System

- [x] **UPDT-01**: User can press [v] on a skill to verify if upstream has newer commits (git hash comparison)
- [x] **UPDT-02**: User can press [u] on a skill to update it from upstream
- [x] **UPDT-03**: User can press [g] to update all installed skills globally
- [x] **UPDT-04**: Skill lock file or config stores git commit hash for installed version tracking

### Doctor

- [x] **DCTR-01**: Doctor command verifies all skills in config exist at skills-path on filesystem
- [x] **DCTR-02**: Doctor detects skills on filesystem not tracked in config and offers to add them
- [x] **DCTR-03**: Doctor backfills metadata for pre-v0.2.0 installed skills using known correspondence list
- [x] **DCTR-04**: Doctor prints alert messages for any problems found

## v2 Requirements

Deferred to future release.

### Notifications

- **NTFY-01**: Show visual indicator in manage view when skill has upstream updates available

### Advanced Update

- **ADVN-01**: Changelog display showing what changed between installed and latest version
- **ADVN-02**: Rollback to previous skill version

## Out of Scope

| Feature | Reason |
|---------|--------|
| Auto-backfill on startup | Only Doctor handles metadata gaps -- keeps startup fast |
| Self-update of efx-skills binary | Separate distribution concern |
| New registry integrations | Beyond skills.sh, playbooks.com, GitHub -- future |
| Authentication/API keys | Not needed for current registries |
| Windows browser open | Focus on macOS/Linux first |
