<objective>
Create the config mutation functions needed for install/uninstall metadata lifecycle: saveConfigData (standalone config persistence), addSkillToConfig, removeSkillFromConfig, and skillMetaFromAPISkill.

Purpose: Phase 2 install/uninstall flows need to write and remove SkillMeta entries from config.json. The existing `configModel.saveConfig()` is tied to the config view's in-memory model and cannot be used from the search or manage views. These standalone functions operate on disk directly via load-modify-save.

Output: Four exported functions in config.go with full test coverage in config_test.go.
</objective>

<feature>
  <name>Config mutation functions for skill metadata lifecycle</name>
  <files>internal/tui/config.go, internal/tui/config_test.go</files>
  <behavior>
    - addSkillToConfig: given a SkillMeta, appends it to the config.json skills array on disk
    - addSkillToConfig: if a SkillMeta with the same Owner AND Name already exists, does NOT create a duplicate (idempotent)
    - addSkillToConfig: if no config file exists on disk, creates one with defaults and the new skill
    - removeSkillFromConfig: given a skill name, removes the matching SkillMeta entry from config.json
    - removeSkillFromConfig: if the skill name is not in the skills array, returns nil (no-op, no error)
    - removeSkillFromConfig: if no config file exists on disk, returns nil (no-op, no error)
    - saveConfigData: writes a *ConfigData to ~/.config/efx-skills/config.json as indented JSON
    - saveConfigData: ensures Skills is [] not null in the output JSON
    - saveConfigData: creates the config directory if it does not exist
    - skillMetaFromAPISkill: constructs a SkillMeta from an api.Skill, using Source as Owner, Name as Name, Registry as Registry, and "https://github.com/{Source}" as URL
  </behavior>
  <implementation>
    Add to internal/tui/config.go:

    1. `saveConfigData(cfg *ConfigData) error` -- standalone config writer. Reads HOME, builds config path, ensures directory, marshals with MarshalIndent, writes file. Ensures Skills is `[]SkillMeta{}` not nil before marshal. Ensures SkillsPath defaults via `defaultSkillsPath()` if empty.

    2. `addSkillToConfig(meta SkillMeta) error` -- loads config from file (or creates empty ConfigData with defaults if nil), checks for duplicate by Owner+Name, appends if not found, calls saveConfigData.

    3. `removeSkillFromConfig(skillName string) error` -- loads config from file, returns nil if nil. Filters cfg.Skills to exclude entries where Name == skillName. Calls saveConfigData with filtered result.

    4. `skillMetaFromAPISkill(s api.Skill) SkillMeta` -- pure function. Maps: Owner=s.Source, Name=s.Name, Registry=s.Registry, URL=fmt.Sprintf("https://github.com/%s", s.Source).

    Import `"github.com/lmarques/efx-skills/internal/api"` in config.go for the api.Skill type reference.

    Tests in config_test.go use temp directories (t.TempDir()) to isolate filesystem operations. Override HOME env var in tests to point to temp dir so loadConfigFromFile/saveConfigData use the test directory. Restore HOME after test using t.Cleanup. Test names:
    - TestAddSkillToConfig (happy path)
    - TestAddSkillToConfigIdempotent (duplicate detection)
    - TestAddSkillToConfigNoExistingFile (creates config from scratch)
    - TestRemoveSkillFromConfig (happy path)
    - TestRemoveNonexistentSkill (no-op)
    - TestSaveConfigDataCreatesDir (directory creation)
    - TestSkillMetaFromAPISkill (field mapping)
  </implementation>
</feature>
