    **manage.go -- removal flow (applySkillChanges function, ~line 387-406):**

    The current `applySkillChanges` function iterates skills and creates/removes symlinks. When a skill is unlinked (`!skill.Selected && skill.Linked`), it calls `os.RemoveAll(linkPath)`. After this removal, the function should check whether the skill is still linked to ANY other provider. If not, remove its SkillMeta from config.

    Replace the existing `applySkillChanges` function body with logic that:
    1. Keeps the existing symlink create/remove loop unchanged
    2. After the loop, collects the names of skills that were unlinked (were Linked=true, now Selected=false)
    3. For each unlinked skill, checks all configured providers (via `detectProviders()`) to see if a symlink still exists for that skill name
    4. If no provider has the skill linked, calls `removeSkillFromConfig(skill.Name)`

    Implementation detail for the "still linked anywhere" check:
    ```go
    // After the symlink create/remove loop:
    allProviders := detectProviders()
    for _, skill := range skills {
        if !skill.Selected && skill.Linked {
            // Skill was unlinked from this provider; check if linked elsewhere
            stillLinked := false
            for _, p := range allProviders {
                if p.Configured {
                    linkPath := filepath.Join(p.Path, skill.Name)
                    if _, err := os.Lstat(linkPath); err == nil {
                        stillLinked = true
                        break
                    }
                }
            }
            if !stillLinked {
                removeSkillFromConfig(skill.Name)
            }
        }
    }
    ```

    Do NOT add the `api` import to manage.go -- it is not needed here since `removeSkillFromConfig` takes a string, not an api.Skill.

This satisfies ORIG-01 as-is. No changes needed. The display format matches the spec: `skill-name   owner/repo   popularity`.
