  <action>
    Add three new message types and keybinding handlers to manage.go:

    1. **Message types** (add near existing skillsLoadedMsg):
       ```go
       type verifySkillMsg struct {
           skillName string
           hasUpdate bool
           currentHash string
           latestHash  string
           err         error
       }

       type updateSkillMsg struct {
           skillName string
           err       error
       }

       type updateAllMsg struct {
           updated []string
           err     error
       }
       ```

    2. **Model fields** -- add to manageModel struct:
       ```go
       statusMsg string  // feedback message shown at bottom of view
       updating  bool    // true while an update operation is in progress
       ```

    3. **Keybinding handlers** in Update() -- add cases in the tea.KeyMsg switch, alongside existing "o" and "s" cases:

       **[v] -- Verify selected skill:**
       - Guard: only when selectedIdx is on a skill (not a group), and not currently updating
       - Set `m.updating = true`, `m.statusMsg = "Checking for updates..."`
       - Return tea.Cmd closure that:
         - Creates a Store via `skill.NewStore(skillsPath)` (get skillsPath from loadConfigFromFile)
         - Calls `store.CheckForUpdate(skillName)`
         - Returns `verifySkillMsg{skillName, hasUpdate, currentHash, latestHash, err}`

       **[u] -- Update selected skill:**
       - Guard: only when selectedIdx is on a skill (not a group), and not currently updating
       - Set `m.updating = true`, `m.statusMsg = fmt.Sprintf("Updating %s...", skillName)`
       - Return tea.Cmd closure that:
         - Creates a Store via `skill.NewStore(skillsPath)`
         - Calls `store.UpdateSkill(skillName)`
         - Returns `updateSkillMsg{skillName, err}`

       **[g] -- Global update all skills:**
       - Guard: not currently updating
       - Set `m.updating = true`, `m.statusMsg = "Updating all skills..."`
       - Return tea.Cmd closure that:
         - Creates a Store via `skill.NewStore(skillsPath)`
         - Calls `store.UpdateAllSkills()`
         - Returns `updateAllMsg{updated, err}`

    4. **Message handlers** in Update() -- add cases for the new message types (alongside existing skillsLoadedMsg case):

       **verifySkillMsg:**
       - Set `m.updating = false`
       - If err != nil: `m.statusMsg = fmt.Sprintf("Error checking %s: %v", msg.skillName, msg.err)`
       - If hasUpdate: `m.statusMsg = fmt.Sprintf("Update available for %s (installed: %.7s, latest: %.7s)", msg.skillName, msg.currentHash, msg.latestHash)`
       - If !hasUpdate: `m.statusMsg = fmt.Sprintf("%s is up to date (%.7s)", msg.skillName, msg.currentHash)`
       - Use `%.7s` to show abbreviated 7-char commit hashes for readability

       **updateSkillMsg:**
       - Set `m.updating = false`
       - If err != nil: `m.statusMsg = fmt.Sprintf("Error updating %s: %v", msg.skillName, msg.err)`
       - If no error: `m.statusMsg = fmt.Sprintf("Updated %s successfully", msg.skillName)`

       **updateAllMsg:**
       - Set `m.updating = false`
       - If err != nil: `m.statusMsg = fmt.Sprintf("Update errors: %v", msg.err)`
       - If len(updated) == 0: `m.statusMsg = "All skills are up to date"`
       - Otherwise: `m.statusMsg = fmt.Sprintf("Updated %d skills: %s", len(msg.updated), strings.Join(msg.updated, ", "))`

    5. **View() update** -- add status message display after the help text (or just before it):
       - If `m.updating`: show `spinnerStyle.Render("  " + m.statusMsg)`
       - Else if `m.statusMsg != ""`: show styled status message
         - If starts with "Error": use `errorStyle`
         - If starts with "Update available": use `statusWarnStyle`
         - If starts with "Updated" or "up to date" or "All skills": use `statusOkStyle`
         - Render as `"  " + m.statusMsg`

    6. **Help text update** -- update the help line in View() to include the new keybindings:
       Change from:
       ```
       [space] preview  [o] open  [t] toggle  ...
       ```
       To:
       ```
       [space] preview  [o] open  [v] verify  [u] update  [g] update all  [t] toggle  ...
       ```

    7. **Import** -- add `"github.com/lmarques/efx-skills/internal/skill"` to manage.go imports (alongside existing imports). It is already imported indirectly but needs direct import for `skill.NewStore()`.

    Helper function to get skillsPath (add as a package-level function or inline):
    ```go
    func getSkillsPath() string {
        cfg := loadConfigFromFile()
        if cfg != nil && cfg.SkillsPath != "" {
            return cfg.SkillsPath
        }
        return defaultSkillsPath()
    }
    ```
