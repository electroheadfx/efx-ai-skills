  <action>
    1. Create internal/tui/doctor_cmd.go with package tui:

    ```go
    func RunDoctor(fix bool) error
    ```

    RunDoctor orchestrates the full doctor flow:
    a. Load config via loadConfigFromFile() -- if nil, use empty defaults
    b. Determine skillsPath via getSkillsPath()
    c. Call RunDiagnostics(skillsPath, cfg.Skills)
    d. Print FormatReport(report) to stdout via fmt.Print
    e. If fix flag is true AND report.BackfillCandidates is non-empty:
       - Call BackfillLegacySkills(report.BackfillCandidates, skillsPath)
       - Print backfill results (e.g., "Backfilled N skills: skill1, skill2")
    f. If report has any error-severity issues, return a non-nil error (so cobra exits with code 1)
       - Use a simple error like: fmt.Errorf("doctor found %d issue(s)", len(report.Issues))
       - If no issues or only warnings/info, return nil

    2. In cmd/efx-skills/main.go, add the doctor command following the existing pattern:

    ```go
    doctorCmd := &cobra.Command{
        Use:   "doctor",
        Short: "Verify and repair skill installation integrity",
        RunE: func(cmd *cobra.Command, args []string) error {
            fix, _ := cmd.Flags().GetBool("fix")
            return tui.RunDoctor(fix)
        },
    }
    doctorCmd.Flags().Bool("fix", false, "Automatically backfill legacy skill metadata")
    ```

    Add doctorCmd to rootCmd.AddCommand(...) alongside existing commands.
